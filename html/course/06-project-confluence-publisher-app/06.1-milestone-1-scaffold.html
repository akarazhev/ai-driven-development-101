Constraints:

- Include: relationships, indexes, constraints
- Use [your ORM/framework]
- Include migration files
Inputs: [Data model from main README]
Output format: Schema definition + migration files
Evaluation: Schema supports all requirements

Tables to create:

  • attachment_assets: id, filename, content_type, size, storage_path, alt_text
  • pages: id, text, created_at, updated_at, author_id
  • page_attachment: id, page_id, attachment_id, position
  • schedules: id, page_id, scheduled_at, status, attempt_count, last_error
  • publish_logs: id, page_id, provider, external_id, status, message, created_at

Verification checklist

Before moving to M2, verify:

Troubleshooting

Issue: Backend won’t start

Solutions:

  • Check dependencies are installed
  • Verify configuration files
  • Check port isn’t in use
  • Review error logs

Issue: Database connection fails

Solutions:

  • Verify database is running
  • Check connection string
  • Verify credentials
  • Check firewall/network

Issue: Frontend can’t reach backend

Solutions:

  • Check CORS configuration
  • Verify backend URL
  • Check network/firewall
  • Review API client setup
